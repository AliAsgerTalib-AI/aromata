# Density Shift Analysis Feature Design

**Date:** 2026-05-26  
**Status:** Design Approved  
**Author:** Claude Code + User  

## Overview

Add a new analytical section to the FragranceDossier that explains how a fragrance distributes its molecular weight between high-volatility (top-note) and low-volatility (base-note) compounds. This section appears as ROW 3.5, positioned after performance metrics (longevity, sillage) and before market analysis.

The section has a family-adaptive title (e.g., "Gourmand Density Shift", "Woody Density Shift") and uses a hybrid approach: Gemini generates narrative descriptions while compound lists are extracted from existing data.

## Design Decisions

### 1. Scope: All Fragrances, Family-Adaptive

The feature applies to **every fragrance**, not just gourmands. The section title and language adapt based on the fragrance's primary olfactory family:
- **Gourmand** → "Gourmand Density Shift"
- **Woody** → "Woody Density Shift"
- **Floral** → "Floral Density Shift"
- **Oriental/Amber** → "Amberic Density Shift"
- **Fresh/Citrus** → "Fresh Density Shift"
- (Pattern: `${familyFocus} Density Shift`)

### 2. Data Generation: Hybrid Approach

**Extracted from existing data:**
- High-volatility compounds: sourced from `aromaChemicalMatrix` entries with short evaporation times (top notes, <1 hour per `evaporationCurve`)
- Low-volatility compounds: sourced from `aromaChemicalMatrix` entries with long evaporation times (base notes, >4 hours per `evaporationCurve`)

**Generated by Gemini:**
- **Diffusion Effect:** Narrative describing the immediate sensory impact of high-volatility compounds
- **Tenacity Effect:** Narrative describing the long-lasting experience created by low-volatility compounds
- **Strategic Portfolio Takeaway:** Strategic insight about formulation choices, market positioning, or house philosophy

**Why hybrid?**
- Compound extraction is deterministic and reliable
- Narratives benefit from Gemini's context-aware language and domain expertise
- Reduces API load compared to asking Gemini to generate everything

### 3. API Integration: Single Gemini Call

The density shift analysis is **bundled into the main fragrance analysis request**, not a separate call. This is achieved by adding `densityShiftAnalysis` to the existing response schema in `buildFragranceDataSchema()`.

**Gemini prompt structure:**
```
You are a fragrance formulation strategist. Analyze the density shift in this [family] fragrance.

Top-note compounds (high volatility): [comma-separated list]
Base compounds (low volatility): [comma-separated list]
Primary accord: [accord name]

For DIFFUSION EFFECT: Describe the immediate sensory impact. What does the top-note engine accomplish?
For TENACITY EFFECT: Describe the long-lasting experience. How do base compounds create the signature wear?
For STRATEGIC PORTFOLIO TAKEAWAY: What is the commercial/formulation insight? Why these choices?

Keep each section to 2-3 sentences. Be authoritative and scientific.
```

### 4. Placement: ROW 3.5 in FragranceDossier

**Location:** After ROW 3 (Longevity & Performance), before ROW 4 (Accords & Settings)

**Rationale:**
- By this point, readers understand the fragrance's chemical composition and performance metrics
- This section bridges technical data (evaporation curves, longevity indices) with strategic narrative
- It prepares readers for market analysis by explaining the formulation philosophy

### 5. Graceful Degradation

- Component renders **only if** `fragrance.densityShiftAnalysis` exists
- If Gemini fails to generate this field, the FragranceDossier still renders completely without it
- No impact on print layout or dossier functionality

## Data Structure

### New TypeScript Interfaces

**In `src/types.ts`** — add to `FragranceData`:
```typescript
densityShiftAnalysis?: DensityShiftAnalysis;
```

**In `src/server/types/analysisTypes.ts`** — new interface:
```typescript
export interface DensityShiftAnalysis {
  familyFocus: string;                    // "Gourmand", "Woody", "Floral", etc.
  highVolatilityEngine: string;           // Key top-note compound names
  diffusionEffect: string;                // Gemini-generated narrative
  lowVolatilityEngine: string;            // Key base compound names
  tenacityEffect: string;                 // Gemini-generated narrative
  strategicPortfolioTakeaway: string;     // Gemini-generated insight
}
```

### Gemini Schema Addition

**In `src/server/gemini/fragranceDataPrompt.ts`**, add to `buildFragranceDataSchema()`:

```typescript
densityShiftAnalysis: {
  type: Type.OBJECT,
  required: [
    'familyFocus',
    'highVolatilityEngine',
    'diffusionEffect',
    'lowVolatilityEngine',
    'tenacityEffect',
    'strategicPortfolioTakeaway'
  ],
  properties: {
    familyFocus: {
      type: Type.STRING,
      description: 'Primary fragrance family (Gourmand, Woody, Floral, Oriental, Fresh, etc.)'
    },
    highVolatilityEngine: {
      type: Type.STRING,
      description: 'Key top-note compounds driving initial diffusion (comma-separated list of compound names)'
    },
    diffusionEffect: {
      type: Type.STRING,
      description: 'Narrative describing the immediate sensory impact and diffusion behavior (2-3 sentences)'
    },
    lowVolatilityEngine: {
      type: Type.STRING,
      description: 'Key base compounds creating longevity and signature wear (comma-separated list of compound names)'
    },
    tenacityEffect: {
      type: Type.STRING,
      description: 'Narrative describing the long-lasting sensory experience (2-3 sentences)'
    },
    strategicPortfolioTakeaway: {
      type: Type.STRING,
      description: 'Strategic insight into formulation choices, market positioning, or house philosophy (2-3 sentences)'
    }
  }
}
```

## Component Design

### `DensityShiftAnalyzer.tsx`

**Purpose:** Render the density shift analysis section in the FragranceDossier

**Props:**
```typescript
interface DensityShiftAnalyzerProps {
  fragrance: FragranceData;
}
```

**Responsibilities:**
1. Extract `densityShiftAnalysis` from `fragrance`
2. Build the section title as `${densityShiftAnalysis.familyFocus} Density Shift`
3. Render four subsections:
   - **High Volatility Scaffolding (Radiative Focus)** with Molecular Engine and Diffusion Effect
   - **Low Volatility Scaffolding (Density Focus)** with Molecular Engine and Tenacity Effect
   - **Strategic Portfolio Takeaway**

**Visual style:**
- Match existing FragranceDossier card styling (dark background, borders, monospace labels)
- Use Lucide icons: `Wind` for high volatility, `Droplet` for low volatility
- Monospace labels (uppercase, small font): `[HIGH VOLATILITY ENGINE]`, `[DIFFUSION EFFECT]`, etc.
- Narrative text in regular sans-serif, lighter color

**Example rendered output:**
```
Gourmand Density Shift

High Volatility Scaffolding (Radiative Focus)
[MOLECULAR ENGINE]
Pear esters and high-dose citrus fractions.

[DIFFUSION EFFECT]
Immediate, sharp edible sweetness that alerts the senses.

Low Volatility Scaffolding (Density Focus)
[MOLECULAR ENGINE]
Ethyl Vanillin paired with dense synthetic musks.

[TENACITY EFFECT]
A long-lasting, skin-hugging sweetness that creates the 'addictive' gourmand trail.

Strategic Portfolio Takeaway
YSL successfully leveraged the 'gourmand' trend by focusing on high-impact, low-cost synthetic materials that offer immense projection and mass appeal, ensuring consistency across reformulated batches while maintaining a premium pricing strategy.
```

## Integration into FragranceDossier

**In `src/components/FragranceDossier.tsx`**, add after ROW 3 (Longevity & Performance):

```typescript
{fragrance.densityShiftAnalysis && (
  <DensityShiftAnalyzer fragrance={fragrance} />
)}
```

This ensures the component only renders when the data exists, with no impact on existing rows or layout.

## Error Handling & Edge Cases

1. **Missing `densityShiftAnalysis`:** Component gracefully doesn't render. Dossier is complete without it.
2. **Empty compound lists:** Display "Not specified" or similar fallback text.
3. **Very short narratives:** No minimum length enforced; Gemini may return 1-sentence descriptions in rare cases. Display as-is.
